# Factory Runtime Deploy Plan

## Deployment Impact

This iteration stays inside the composer-adjacent footer presentation and verification layers. The bounded expectation is that the bottom-fixed composer now carries one compact selected-thread target-and-transport strip that agrees with the canonical ownership source and downgrades or clears immediately on degraded or switched paths.

## Rollout Notes

1. Apply the proposal commit onto `main`.
2. Enable `CODEX_FACTORY_ENABLE_INTERNAL_APPEND_SSE=1` only in the internal runtime where the workspace should consume live append frames.
3. Open the operator console on desktop and phone widths with a healthy selected-thread live session.
4. Confirm the composer-adjacent strip stays mounted with the footer and shows the selected thread target plus `READY` and `SSE OWNER` on the healthy selected-thread path.
5. Trigger reconnect or polling fallback and confirm the footer strip downgrades to `RECONNECT` or `POLLING` without pretending the composer still owns a healthy live session.
6. Switch intentionally to another thread and confirm the footer strip stays mounted, changes to `SWITCHING` plus `ATTACH`, and clears old selected-thread ownership on the same transition.
7. Run `BASE_URL=... API_KEY=... WORKSPACE_APP_ID=factory-runtime ./scripts/verify_deployed_console.sh` and confirm header, transcript, rail, and composer-adjacent strip all agree on one canonical ownership source through the intended browser path.
