# Factory Runtime Deploy Plan

## Deployment Impact

This iteration stays inside the selected-thread header and composer-strip presentation plus verification layers. The bounded expectation is that healthy selected-thread SSE ownership now spells out the current autonomy phase directly on the existing header and composer-adjacent surfaces, while degraded states remain clearly non-owned.

## Rollout Notes

1. Apply the proposal commit onto `main`.
2. Enable `CODEX_FACTORY_ENABLE_INTERNAL_APPEND_SSE=1` only in the internal runtime where the workspace should consume live append frames.
3. Open the operator console on desktop and phone widths with a healthy selected-thread live session.
4. Start a healthy selected-thread SSE-owned run and confirm the header badge moves through explicit PROPOSAL, REVIEW, VERIFY, AUTO APPLY, READY, or APPLIED labels with short detail copy.
5. Confirm the composer-adjacent strip stays mounted, keeps `READY` plus `SSE OWNER`, and adds the same explicit phase chip with concise detail copy on the healthy path.
6. Trigger reconnect or polling fallback and confirm the header and composer strip drop back to degraded non-owned labels instead of pretending the session is still healthy-live.
7. Switch intentionally to another thread and confirm phase-specific healthy labels clear with the ownership handoff instead of leaking into the new selection.
8. Run `BASE_URL=... API_KEY=... WORKSPACE_APP_ID=factory-runtime ./scripts/verify_deployed_console.sh` and confirm the browser-visible workspace proves exact healthy selected-thread phase progression through the existing header and composer surfaces.
